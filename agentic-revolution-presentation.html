What Is an AI Agent?

Definition

An AI agent perceives its environment (sensors), reasons, plans, and acts (actuators) autonomously to achieve goals (Russell & Norvig).

PEAS Framework

  • Performance Measure: Success metric (e.g., task accuracy, speed)
  • Environment: Digital platforms, APIs, databases
  • Actuators: Tools like API calls, generative outputs
  • Sensors: Inputs like text, RAG, or MCP data

Components

  • Brain: LLM (e.g., Claude, GPT-5) for reasoning
  • Sensors: RAG for knowledge retrieval, MCP for live data
  • Actuators: Function/tool calling, MCP for actions
  • Memory: Short-term (context window), long-term (RAG)

Model Context Protocol (MCP)

Definition

MCP standardizes agent-to-tool communication, enabling dynamic interactions with APIs, files, or systems (Anthropic, 2025).

Components

  • Tools: Functions agents can execute
  • Resources: Data agents can access
  • Prompts: Instructions for agents

Best Practices for Prompting AI Agents

Five Fundamentals (Rick Mulready, 2025)

  • 1. Specificity: Define format, length, structure (e.g., XML tags)
  • 2. Context: Include audience, goal, voice
  • 3. Few-Shot Prompting: Provide examples for pattern matching
  • 4. Structuring: Use CO-STAR (Context, Objective, Style, Tone, Audience, Response)
  • 5. Iteration: Refine prompts, ask AI to self-critique (e.g., 10/10 score)

Agent Prompt Structure

Role/Objective: Define the agent's purpose
Operating Instructions: Step-by-step process
Constraints: Limits and boundaries

Stress iterative prompting to unlock agent potential.

Multi-Agent Systems: Power of Collaboration

Why MAS?

Overcomes single LLM limitations, enables specialization, improves memory via RAG.

Patterns

Sequential
Tasks flow linearly (e.g., Searcher ‚Üí Summarizer)
Hierarchical
Orchestrator delegates to sub-agents
Parallel
Sub-agents work simultaneously
Critique/Debate
Agents review outputs for quality
Compare MAS to human teams for efficiency gains. Each agent specializes, enabling complex workflows.

Agents.md: Standardizing Workflows

Definition

Markdown-based standard for project-wide rules (e.g., test commands, file structure).

# Agents.md

## Project Rules
- Always run tests before committing
- Use TypeScript for all new files
- Follow the established folder structure
- Document all API endpoints

## Agent Instructions
- Use RAG for company knowledge
- Validate all outputs before execution
- Request approval for destructive actions

Purpose: Ensures interoperability and consistency.

Use Case: Governs Claude Code workflows, standardizing repository interactions.

Benefit: Acts as a "README for agents," reducing setup errors and enabling scalable systems.
