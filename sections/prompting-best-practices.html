Prompting Best Practices

200 characters can be the difference between a brilliant agent and a frustrating one. Design prompts that communicate role, intent, process, and output.

Opening Hook

“The difference between a helpful agent and a frustrating one often comes down to 200 characters in a prompt.”

The Five Essential Elements

  • Role & context: “You are a financial analyst at MegaCorp...”
  • Objective: State success criteria explicitly.
  • Constraints: Guardrails around tone, data sources, length.
  • Process: Step-by-step instructions to reduce errors.
  • Output format: Define JSON, tables, or other structured responses.

Email Triage Evolution

  1. V1: “Sort my emails.” → inconsistent results.
  2. V2: “Categorize emails as urgent/action/info/spam.” → better, but subjective.
  3. V3: Detailed definitions, extraction requirements, and JSON output → 91% accuracy.

Tool Usage Schema Example

You have access to web_search(query: string) → [results]
Best practices:
- Use specific queries
- Search one topic at a time
- Rephrase if the first search fails
- Cite sources in final answer

Explicit schemas lift tool success from 61% to 84% in testing.

Chain-of-Thought Prompting

Calculate 15% of $2,847.
Show your work step-by-step before the final answer.

Encouraging deliberate reasoning reduced arithmetic errors by 68% in pilot runs.

Structured Outputs

Summarize these articles in JSON:
{
  "articles": [{
    "title": "...",
    "source": "...",
    "key_point_1": "...",
    "key_point_2": "...",
    "relevance_score": 1-10
  }],
  "overall_theme": "...",
  "contradictions": []
}

Structured output requests boost downstream automation success by 79%.

Common Mistakes & Fixes

Mistake Why It Fails Fix
“Be creative” Too vague Describe the type of creativity you expect.
Negative framing (“Don't make mistakes”) Doesn't guide behavior State positive validation steps.
Overloaded prompts Agents lose focus Prioritize top three instructions and move examples to references.
No examples Agent guesses intent Add two or three exemplars of ideal outputs.

Proof Points

Step-by-step prompts reduce errors by 43% and adding just two examples improves accuracy 15–25%.
