BMAD Method — Structured Agent Collaboration

Assign agents to the same roles humans use—Business Analyst, Model PM, Architect, Developer—for fast, predictable delivery.

Opening Hook

“Software development has roles for a reason. So should your agent team.”

Role Outputs

Business Analyst

Translates fuzzy ideas into problem statements, success metrics, constraints, stakeholders, and open questions.

Product Manager

Produces prioritized user stories, hypotheses, and acceptance criteria.

Architect

Designs system components, data flows, integrations, and risk mitigations.

Developer

Implements code, tests, prompts, and workflows according to the design.

Artifact Examples

## Business Requirements
- Problem: Customer churn 8% (target 6%)
- Success: Reduce churn 2% in 6 months
- Constraints: Budget $50K, timeline Q2

## Product Requirements
- Story: Identify at-risk customers
- Acceptance: 90% prediction accuracy

## Architecture
- Components: Churn model, email generator, campaign manager
- Data Flow: Activity → Model → Score table → Email service

Cycle Cadence

Week 1: BA → PM → Architect → Developer. Week 2: Measure. Week 3: BA reviews data, iteration begins anew. Total cycle time: 7–10 days instead of 3–4 weeks.

Map to Existing Rituals

  • Agents prepare artifacts before sprint planning and refinement.
  • Humans prioritize, unblock, and review outputs.
  • Retrospectives include agent performance insights.

Documentation by Default

Each handoff produces a document. Human checkpoints confirm accuracy before the next role proceeds.

Cycle Metrics

Stage Target Time
Idea → BA Doc < 1 day
BA Doc → PRD < 1 day
PRD → Architecture < 2 days
Architecture → Working Code < 5 days

Payment Retry Logic
