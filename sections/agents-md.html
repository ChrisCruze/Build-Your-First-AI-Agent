Agents.md Standard — The Team Playbook

Documentation is the shared language that keeps humans and agents aligned. Agents.md turns tacit knowledge into repeatable instructions.

Opening Hook

“Documentation isn't bureaucracy—it's the shared language that lets humans and agents collaborate without chaos.”

Why Agents.md?

  • Eliminates guesswork when agents enter a repo.
  • Makes conventions and guardrails explicit.
  • Serves as onboarding for both humans and machines.

Suggested Structure

# Agents.md - [Project]
## Purpose
## Agent Instructions
### Code Conventions
### Testing Requirements
### Agent Capabilities
## Human Guidelines
### Getting Started
### PR Process
### Escalation

Example Snippet

## Agent Instructions for Checkout Flow
- NEVER disable payment validation
- ALWAYS require human review for discount logic
- Tests: `npm test` + staging checkout run

Repository Conventions

  • Consistent file naming (kebab-case JS, snake_case Python).
  • Directory patterns (components/, services/, utils/).
  • Testing rules (unit + integration before merging).

Keep It Current

  • Weekly skim to capture emerging patterns.
  • Monthly deep review with the team.
  • Update immediately after major architectural or process changes.

Onboarding Advantages

New humans ramp quickly by reading Agents.md. New agents produce code that matches house style from their first PR.

Impact Metrics

Teams with Agents.md reduce agent surprise behaviors by 78% and shrink agent onboarding time from two days to two hours.
